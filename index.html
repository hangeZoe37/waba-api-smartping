WhatsApp Business API

The SmartPing WABA API lets you create message templates, run bulk campaigns, send individual transactional messages, track delivery, upload files, and manage consent — all over a single REST API. This reference covers every endpoint, every parameter, and the exact behaviour you can expect.

Base URL https://ocmpapi.sparc.smartping.io/backend/api
ℹ Every request requires an Authorization: Bearer <token> header — except GET /campaign/health. Tokens expire after 3,600 seconds (1 hour). See Authentication for how to handle expiry.

Quickstart — Send Your First Bulk Campaign

This guide walks you through the complete bulk campaign flow from scratch. You'll need your account_id, bsp_id, and waba_number — all available in the OCMP dashboard under Settings → Account.

ℹ The bulk campaign flow has exactly three steps: Upload your CSV → Save the campaign → Send it. The save step triggers background preprocessing (validation, deduplication, blocked-number removal). You then explicitly trigger dispatch when ready.

Step-by-step

1
Verify your credentials
Call the credentials endpoint to confirm your account is active and retrieve your identifiers. You'll use account_id, bsp_id, and waba_number in nearly every subsequent request.
curl 'https://ocmpapi.sparc.smartping.io/backend/api/licenses/swagger/credentialsdata' \
  -H 'Authorization: Bearer <your-token>'
2
Confirm your template is approved
You can only send campaigns using templates with status: "APPROVED". List your templates and save the full template object — you'll pass the entire object (not just the ID) as template_data when creating a campaign.
curl 'https://ocmpapi.sparc.smartping.io/backend/api/templates/{account_id}/{bsp_id}/{waba_number}?page=1&size=50' \
  -H 'Authorization: Bearer <your-token>'

# Filter the response for status === "APPROVED" before proceeding
3
Upload your recipient CSV
Upload a CSV file where each row is a phone number in full international format without the + prefix (e.g. 919876543210). The response returns a url — save this as your bulk_upload_url.
⚠ Phone numbers must include the country code. Numbers that are malformed or already blocklisted are silently dropped during preprocessing — they will not cause the campaign to fail, but they will not receive messages. 
curl -X POST 'https://ocmpapi.sparc.smartping.io/backend/api/drive/upload' \
  -H 'Authorization: Bearer <your-token>' \
  -F 'file=@/path/to/recipients.csv' \
  -F 'account_id=<account_id>' \
  -F 'channel=waba' \
  -F 'created_by=My App'

# Response — save the "url" field
{
  "url": "https://ocmp.smartping.io/backend/api/drive/uploads/<account_id>/<filename>.csv"
}
4
Save the campaign
Create the campaign by combining your approved template object, CSV URL, and account details. Setting is_drafted: false immediately queues preprocessing. The response returns the campaign_id you'll need for the next steps.
⚠ Pass the complete template object from GET /templates as template_data — not just the template ID. The campaign manager needs the full object including bsp_metadata and bsp_template_id. Passing an incomplete object silently fails preprocessing. 
curl -X POST 'https://ocmpapi.sparc.smartping.io/backend/api/campaign/save-campaign' \
  -H 'Authorization: Bearer <your-token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "account_id": "<account_id>",
    "channel": "waba",
    "waba_number": "<waba_number>",
    "bsp_id": "<bsp_id>",
    "campaign_name": "My First Campaign",
    "created_by": "My App",
    "template_data": <full-template-object-from-step-2>,
    "bulk_upload_url": "<csv-url-from-step-3>",
    "is_bulk_campaign": true,
    "is_drafted": false
  }'

# Response — save the campaign "id"
{ "id": "9af51935-cc3b-4bc0-8bc6-4a7bc86827c2" }
5
Poll for VALIDATED status, then send
Preprocessing typically takes 30 seconds to 5 minutes. Poll GET /campaign/{campaign_id} until status equals VALIDATED. Then call send-campaign to dispatch.
🚫 This cannot be undone. Once you call send-campaign, messages begin dispatching immediately to Meta's WhatsApp API. Always test with a small list before sending to your full audience. 
# 1. Poll until VALIDATED
curl 'https://ocmpapi.sparc.smartping.io/backend/api/campaign/<campaign_id>' \
  -H 'Authorization: Bearer <your-token>'

# 2. When status is "VALIDATED", trigger send
curl -X POST 'https://ocmpapi.sparc.smartping.io/backend/api/campaign/send-campaign' \
  -H 'Authorization: Bearer <your-token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "id": "<campaign_id>",
    "bulk_upload_url": "<csv-url-from-step-3>",
    "is_send_to_non_consent_recipients": false
  }'
6
Monitor delivery
Use the analytics endpoints to track delivery rates. For individual message status, use GET /analytics/view-analytics-by-message-id/{message_id}. For campaign-level aggregates, use GET /analytics/view-analytics/{account_id}.

Authentication

The WABA API uses Bearer token authentication. Every request — except the health check — must include a valid JWT in the Authorization header.

Sending an authenticated request

Include the token as a header on every request:

Authorization: Bearer eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9…

Token expiry

Tokens expire after 3,600 seconds (1 hour). An expired token returns 401 Unauthorized. In long-running processes — background workers, scheduled jobs, server applications — implement automatic re-authentication by catching 401 responses, refreshing the token, and retrying the original request.

⚠ Do not cache a token without tracking its expiry time. An approach like storing the token alongside its expiry = Date.now() + 3_600_000 and proactively refreshing before expiry avoids failed requests in production.

let token = null;
let tokenExpiry = 0;

async function getToken() {
  if (Date.now() < tokenExpiry) return token;
  // replace with your auth mechanism
  const res = await fetch('…/licenses/swagger/credentialsdata', {
    headers: { Authorization: `Basic ${btoa('user:pass')}` }
  });
  token = (await res.json()).access_token;
  tokenExpiry = Date.now() + 3_500_000; // refresh 100s before expiry
  return token;
}

async function apiFetch(url, opts = {}) {
  const t = await getToken();
  return fetch(url, {
    ...opts,
    headers: { Authorization: `Bearer ${t}`, 'Content-Type': 'application/json', ...opts.headers }
  });
}

POST /templates/

Create Template

Submit a new WhatsApp message template to SmartPing, which forwards it to Meta for approval. A template defines the fixed structure of a message — including the header, body copy, footer, and any interactive buttons. Templates with dynamic content use numbered placeholders like {{1}}, {{2}} that get substituted at send time.

POST https://ocmpapi.sparc.smartping.io/backend/api/templates/
ℹ Meta approval typically takes a few minutes but can take up to 24 hours. After submitting, poll GET /templates and wait for status: "APPROVED" before using the template in a campaign. Template names must be unique per WABA number.

POST /campaign/send-campaign

Send Campaign

Trigger the dispatch of a preprocessed bulk campaign. Call this after polling GET /campaign/{campaign_id} and confirming the status is VALIDATED. Once called, messages begin reaching recipients immediately.

POST https://ocmpapi.sparc.smartping.io/backend/api/campaign/send-campaign
🚫 Irreversible. Calling this endpoint dispatches messages to Meta's WhatsApp API immediately. There is no pause or cancel. Always test with a small recipient list before sending to your full audience.

Request body · application/json

idstring · uuid
required
The campaign UUID returned when the campaign was created with POST /campaign/save-campaign.
bulk_upload_urlstring · url
required
The recipient CSV URL. Must be the same URL used when the campaign was saved.
is_send_to_non_consent_recipientsboolean
required
false — send only to numbers that have a registered opt-in consent record in the platform. true — send to all numbers in the CSV regardless of consent status.
Use false in production unless you manage consent entirely outside the platform. Sending to non-consented numbers may violate WhatsApp's Business Policy and can trigger account restrictions or bans.

Responses

200Campaign queued for dispatch. Monitor progress with GET /campaign/{campaign_id} — status moves from SENDING to COMPLETED.
400Campaign not yet preprocessed (status is not VALIDATED), or malformed payload.
404Campaign not found in this account's scope.

Campaign Lifecycle

Every bulk campaign moves through a defined sequence of states from creation to delivery completion. Understanding each state tells you which API call to make next and how to recover when something goes wrong.

State transitions

DRAFT
via save-campaign with is_drafted: true
Saved locally without triggering preprocessing. You can freely edit the template, replace the CSV, or change the campaign name. No background processing occurs.
SAVED
via save-campaign with is_drafted: false
Preprocessing queued. The platform begins downloading the recipient CSV. Monitor with GET /campaign/{id}.
PREPROCESSING
automatic — triggered by SAVED
Phone numbers being validated, deduplicated, and filtered. Blocked numbers and invalid formats are removed. Typically 30 seconds to 5 minutes depending on list size. Do not call send-campaign during this state.
VALIDATED
automatic — triggered by successful PREPROCESSING
Recipient list is ready. The campaign will not dispatch until you explicitly call POST /campaign/send-campaign. This is your last opportunity to review before committing.
SENDING
via send-campaign
Messages being dispatched to Meta's WhatsApp API. Monitor with GET /campaign/{id}.
COMPLETED
automatic — terminal state
All messages dispatched. Use the analytics endpoints to review delivery and read rates.

Failure states

StateCauseRecovery
PREPROCESSING_FAILEDCSV URL inaccessible, file malformed, or transient network error during download.Fix the URL or file, then re-save with is_drafted: false. Contact support if the URL is correct and the error persists.
SEND_FAILEDInfrastructure failure during message dispatch.Re-call POST /campaign/send-campaign with the same campaign ID. Already-sent messages are not re-sent.

Error Codes

The WABA API uses standard HTTP status codes. Every error response includes a JSON body with a machine-readable error field and a human-readable message. Implement explicit handling for the codes below.

⚠ For 401: your token has expired. Re-authenticate immediately and retry the original request. In long-running processes, implement automatic token refresh — see Authentication.
CodeMeaningDescription & recommended action
200 OK Request succeeded. Some operations return partial failure details inside a 200 response — for example, skipped consent records. Always check the response body.
400 Bad Request Missing required field or invalid value. Read the message field for specifics. Common causes: incomplete template_data, inaccessible bulk_upload_url, passing a string where a boolean is expected.
401 Unauthorized Token missing, expired, or malformed. Re-authenticate and retry. Check for extra whitespace around the token in the header value. See Authentication.
404 Not Found Resource not found within the scope of the authenticated account. IDs are account-scoped — a valid UUID from another account returns 404. Double-check the ID belongs to the correct account.
409 Conflict A resource with this identifier already exists. For templates: the template name is already taken for this WABA number, even if the original was rejected. Choose a different name.
500 Server Error Unexpected platform error. Retry with exponential backoff starting at 30 seconds. If the error persists beyond 5 minutes, contact SmartPing support with your campaign_id and the exact timestamp.
503 Unavailable Campaign service temporarily offline. Do not retry send operations. Poll GET /campaign/health and resume only after it returns 200.
